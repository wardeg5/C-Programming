CS 133U Program Grading Criteria

Introduction

This document describes the grading criteria used for all programs submitted in this class.  Each program assignment is worth some number of points, and the sections below describe how those points are awarded (or deducted).

Grading Criteria

  • Correctness (50% of total)
    • Your program must compile and link without errors (using gcc C options, not C++. So don't use stream.h or any other C++ libraries)
    • Your program must accept input in the specified format
    • Your program must produce the correct output
    • Your program must conform to the class coding standards
  • Maintainability (25% of total)
    • Your program must be properly document (program header, etc)
    • Your source code file must use the correct file name and format
  • Reliability (10% of total)
    • Your program must produce the correct output given different values of input
  • Efficiency (10% of total)
    • Your program must not contain any unused variables
    • Your program must not contain unreachable code
  • Portability (5% of total)
    • Your program must not use any system-specific features, even if they are included in the standards

How Your Program Is Graded

All assignments must be submitted to the appropriate assignment dropbox (email or hardcopy is not accepted).

Feedback and a grade is forwarded to you in D2L in the dropbox section for that assignment.

  • Do not add extra input or change the type of the input from what is mentioned in the assignment specification. I (auto) grade your assignments using a script which uses gcc compiler commands.
    • I feed it an input file with as many input in the type as mentioned in the assignments.
    • So when you add extra, it upsets my script. You can play with your code and experiment but clean it up prior to submitting.
